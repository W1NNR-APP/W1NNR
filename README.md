# The Competitive Loop (Core Logic)

1. Player creates match (entry fee + time).
2. Opponent joins.
3. Both deposit stake.
4. Funds lock in smart contract.
5. Players check in.
6. Players compete in the real world.
7. Both report outcome.
8. Contract settles automatically.

If both report same winner → instant payout.
If disagreement → dispute logic triggers.
If no show → automatic handling.

No platform custody.
No admin overrides.
No subjective moderation.

Deterministic execution.

# Economic Design

Each match:
- 2 players stake entry fee
- Funds locked
- Winner receives ~92% of pot
- Platform earns small % fee

Why this works:

- Stakes filter unserious players
- Higher stakes signal confidence
- Winning builds reputation
- Reputation attracts higher stake matches
- Economic gravity builds around skill

The better you are, the more opportunity you attract.

# Technical Overview (High-Level)

Smart Contracts:
- MatchFactory: deploys new escrow instances
- MatchEscrow: handles lifecycle
- ERC20 staking token (W1T for MVP)
- Deterministic state machine (Open → Funded → InProgress → Reported → Settled)

Security Principles:
- No admin settlement override
- Funds only move by state rules
- Mutual confirmation required
- Deterministic expiry logic
- Transparent event logs

Frontend:
- Wallet integration (MetaMask)
- PulseChain network enforcement
- Match discovery
- Profile caching (localStorage for MVP)
- Event-based leaderboard aggregation

Design Philosophy:
Keep contracts minimal.
Keep state transitions strict.
Keep custody non-existent.
